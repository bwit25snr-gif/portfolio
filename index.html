1. What is Full Stack Development?

A website that does anything more than show fixed text is really two products glued together: something that runs in the browser, and something that runs on a server somewhere else. A full stack developer is comfortable building and connecting both halves.

The front end (or "client side") is everything the browser downloads and runs: HTML for structure, CSS for appearance and JavaScript for behaviour — often combined with a framework like Bootstrap to speed things up. The back end (or "server side") is the part nobody sees: it receives requests, runs the application's logic, checks who's allowed to do what, and reads from or writes to a database.

The panel opposite shows how the three layers talk to each other. Click each one to see what it's responsible for and which tools are typically used to build it.

Client (the browser)

Everything the user sees and clicks. The browser downloads HTML, CSS and JavaScript and turns them into the page on screen. A framework like Bootstrap provides ready-made styles and components, so this layer doesn't have to be built from scratch.

2. HTML — Structure and Meaning

HTML is the skeleton of a page — it groups content into meaningful pieces and gives the browser (and anyone using a screen reader) clues about what each piece is for.

Semantic elements like <header>, <nav>, <main>, <article>, <aside> and <footer> do the same visual job as a plain <div>, but they also say what the content is. That matters for accessibility, for search engines, and for anyone reading the code later — including future me.

What each tag means
<header>
Introductory content for the page or a section — often a logo, title and navigation.
<nav>
A block of links used for navigating the site.
<main>
The primary content of the page — there should only be one per page.
<article>
A self-contained piece of content, such as a blog post.
<aside>
Content related to, but separate from, the main content.
<footer>
Closing content for the page or section — copyright, contact info, etc.

3. CSS — Layout and Appearance

CSS controls layout and appearance. Every element on a page is a box — the box model (content, padding, border and margin) determines how big that box is and how much space it takes up around itself.

Flexbox is one of the most useful layout tools: it lets a container arrange its children in a row or column and decide how they're spaced out, using just a couple of properties. The panel opposite is a live example — change flex-direction and justify-content with the buttons and watch the boxes (and the CSS) update. The smooth movement is a CSS transition, not JavaScript animation.

4. Bootstrap — A Framework of Ready-Made Parts

Bootstrap is a CSS (and JavaScript) framework: a library of ready-made styles, layout tools and components, so common interface patterns don't need to be built from zero.

Its grid system splits each row into 12 columns. A class like col-md-4 means "take up 4 of those 12 columns — one third of the row — on medium screens and larger, but go full width below that." Combined with components like cards and accordions, a responsive layout can come together very quickly.

In fact, this whole page is built with Bootstrap: the navigation bar at the top, the responsive grid behind every two-column section, the buttons in the CSS playground, and the cards and accordion opposite all come from Bootstrap's classes — none of it is custom-built.

5. JavaScript — Behaviour and Interaction

JavaScript is what makes a page interactive after it's loaded: it listens for events — clicks, typing, submitting a form — and updates the page in response, without needing to reload.

The to-do list opposite is a small example of DOM manipulation. Typing a task and clicking "Add" creates a new list item with document.createElement, clicking the text toggles a "done" style with classList.toggle, and the × button removes the item with remove(). form.addEventListener('submit', (event) => {
  event.preventDefault();
  const li = document.createElement('li');
  const span = document.createElement('span');
  span.textContent = input.value;
  li.appendChild(span);
  list.appendChild(li);
  input.value = '';
});

list.addEventListener('click', (event) => {
  if (event.target.tagName === 'SPAN') {
    event.target.parentElement.classList.toggle('done');
  }
  if (event.target.classList.contains('todo-remove')) {
    event.target.closest('.todo-item').remove();
  }
});
